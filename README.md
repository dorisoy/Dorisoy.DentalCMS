# Dorisoy.DentalCMS

  一款专为私人牙科诊所和商业牙科连锁机构设计集多功能于一体的综合牙科业务管理平台。涵盖了患者信息管理、医生资源管理、药房库存管理（包括产品类别、产品信息、销售开单、发票管理和报表生成）、治疗流程管理（如牙周筛查、患者列表和报表）、处方信息管理、费用科目及凭证管理（包括科目管理、凭证信息、凭证列表和报表），设备管理、分析决策、营销运营、以及系统级的临床信息管理、用户管理和权限控制等功能，以实现数字化赋能的业务落地，并提升业务的运营效率和竞争力。

##  功能需求
1.1 患者管理
    - 患者信息：记录患者的基本信息（如姓名、年龄、联系方式、病史等），支持添加、编辑和查询患者资料。
    - 患者列表：提供所有患者的概览，支持按姓名、就诊日期、病情状态等条件筛选和排序，便于快速定位特定患者。
1.2 医生管理
    - 医生信息：存储医生的个人信息、专业技能、排班情况等，帮助诊所合理安排医疗资源。
    - 医生列表：展示所有医生的简介，便于患者预约和内部调度。
1.3 药房管理
    - 产品类别：分类管理牙科所需的各类药品、耗材，便于快速查找和统计。
    - 产品信息：详细记录每种产品的库存量、进货价格、销售价格等信息，支持批量导入和导出。
    - 订单开单：快速生成药品销售单，记录患者购买信息，支持打印和保存。
    - 发票管理：自动生成发票，符合税务要求，支持电子发票和纸质发票的打印。
    - 库存管理：系统可以对诊所内的药品、医疗器械等商品进行订单管理、库存查询和预警。当库存量低于安全库存时，系统会自动提醒采购，防止缺货现象的发生。同时，系统还可以对商品的采购、销售、退货等流程进行精细化管理，提高库存周转率。
    - 报表：生成库存报告、销售报告等，帮助诊所进行成本控制和库存优化。
1.4 治疗诊断
    - 牙周筛查：提供牙周病筛查模板，记录患者的筛查结果，自动生成治疗建议。
    - 患者列表：按治疗进度、状态分类显示患者，便于跟踪治疗情况。
    - 报表：生成治疗报告，分析治疗效果，支持数据导出，便于与第三方机构分享或用于科研。
1.5 处方管理
    - 处方信息：详细记录医生的处方内容，包括药品名称、剂量、用法等。
    - 处方列表：汇总所有处方，支持按日期、医生、患者等条件查询，便于追踪和审核。
1.6 费用管理
    - 科目管理：定义各种治疗项目和药品的费用科目，支持自定义价格。
    - 凭证信息：记录患者的缴费信息，包括缴费金额、时间、方式等。
    - 凭证列表：展示所有缴费记录，支持查询和打印收据。
    - 报表：生成费用报告，分析诊所的收入构成，为决策提供依据。
1.7 设备管理
    - 设备管理：系统可以对诊所内的各种设备进行登记、分类、编号以及状态跟踪。这有助于诊所实时了解设备的数量、位置、使用状态以及维护情况，确保设备的正常运行和及时维修。
    - 考勤与排班：系统可以记录员工的上下班时间、请假情况等信息，帮助诊所实现员工的考勤管理。此外，系统还可以根据诊所的运营情况和员工的能力特点，进行合理的排班安排，确保诊所的正常运营。
1.8 运营营销
    - 预约挂号：系统支持线上预约挂号功能，患者可以通过手机、微信小程序等渠道预约就诊时间，实现线上线下无缝对接。这不仅可以提高就诊效率，还可以优化诊所的资源配置，提升患者满意度。
    - 电子病历：系统可以实现病历的数字化管理，减少纸质病历的使用，节省存储空间，便于管理和检索。同时，电子病历还可以采用加密技术，保障数据安全。
    - 营销工具：系统内置了多种营销工具，如积分商城、分销工具、渠道裂变管理、请客卡、多人拼团、团购套餐等。这些工具可以帮助诊所进行客户运营和裂变，提升整体业绩和客源的增长。
1.9 分析决策
    - 数据统计分析：系统可以清晰快捷地统计诊所所有的数据，如客户增长数据、销售数据、访客数据以及营销活动数据等。这些数据可以为诊所提供经营分析的依据，帮助诊所掌握目前的经营状况。
    - 报表生成与导出：系统可以自动生成各种报表，如财务报表、库存报表、销售报表等。这些报表可以方便地导出为Excel等格式，供诊所进行进一步的分析和处理。
1.10 系统管理
    - 临床信息：管理临床数据库，确保数据安全、完整和准确。
    - 用户管理：添加、删除、修改用户账户，分配角色和权限，确保系统安全。
    - 权限控制：根据不同角色（如医生、护士、收银员、管理员）设置不同的访问权限，保证数据的隐私性和系统的正常运行。
    - 系统安全与备份：系统采用先进的加密技术和安全措施，确保数据的安全性和完整性。同时，系统还具备数据备份和恢复功能，以防止数据丢失或损坏。
    - 系统升级与维护：系统支持在线升级和维护功能，确保软件始终保持最新版本和最佳性能。同时，系统还提供了完善的售后服务和技术支持，帮助诊所解决使用过程中遇到的问题。

## 业务规则分析
1. 患者管理
1.1 患者信息
    1. 患者ID，患者姓名，监护人，地址，性别，联系电话，年龄，介绍人，职业，日期，头像
    - 前置：
      - 无特定前置条件，但需确保系统已启动并正常运行。
    - 输入：
      - 患者ID（如为自动生成，则无需输入）、患者姓名、监护人、地址、性别、联系电话、年龄、介绍人、职业、日期、头像等。
    - 输出：
      - 患者信息成功添加至系统，生成患者信息页面，可查看患者详细信息。
      - 可生成患者报表，包含患者基本信息及添加时间等。
1.2 患者列表
    1. 搜索，修改患者，删除患者
    - 前置：
      - 已存在患者信息。
    - 输入：
      - 搜索关键词（如患者姓名、联系电话等）。
      - 修改患者信息时，需输入新的患者信息。
      - 删除患者时，需确认删除操作。
    - 输出：
      - 患者列表页面，显示所有或符合条件的患者信息。
      - 修改后的患者信息页面。
      - 删除成功提示，或删除失败原因。
  
2. 医生管理
2.1 医生信息
    1. 医生ID（自动生成），医生姓名，地址，专长，联系电话，资格，开始接诊时间
    - 前置：
      - 无特定前置条件。
    - 输入：
      - 医生ID（自动生成，无需输入）、医生姓名、地址、专长、联系电话、资格、开始接诊时间等。
    - 输出：
      - 医生信息成功添加至系统，生成医生信息页面，可查看医生详细信息。
      - 可生成医生报表，包含医生基本信息及接诊时间等。
2.2 医生列表
    1. 搜索，修改医生，删除医生
    - 前置：
      - 已存在医生信息。
    - 输入：
      - 搜索关键词（如医生姓名、专长等）。
      - 修改医生信息时，需输入新的医生信息。
      - 删除医生时，需确认删除操作。
    - 输出：
      - 医生列表页面，显示所有或符合条件的医生信息。
      - 修改后的医生信息页面。
      - 删除成功提示，或删除失败原因。
3. 药房管理
3.1 产品类别
    1. 添加类别，类别列表
    - 前置：
      - 无特定前置条件。
    - 输入：
      - 类别名称。
    - 输出：
      - 类别成功添加至系统，生成类别列表页面。
3.2 产品信息
    1. 用于添加产品：产品ID，产品名称，条码，属性，单位，选择类别，数量，生产日期，采购价，过期时间，销售价，产品图片。
    - 前置：
      - 已存在产品类别。
    - 输入：
      - 产品ID（如为自动生成，则无需输入）、产品名称、条码、属性、单位、选择类别、数量、生产日期、采购价、过期时间、销售价、产品图片等。
    - 输出：
      - 产品信息成功添加至系统，生成产品信息页面，可查看产品详细信息。
3.3 产品列表
    1. 用于搜索，修改产品，删除产品
    - 前置：
      - 已存在产品信息。
    - 输入：
      - 搜索关键词（如产品名称、条码等）。
      - 修改产品信息时，需输入新的产品信息。
      - 删除产品时，需确认删除操作。
    - 输出：
      - 产品列表页面，显示所有或符合条件的产品信息。
      - 修改后的产品信息页面。
      - 删除成功提示，或删除失败原因。
3.4 销售开单
    1. 用于新增销售开单，患者编号, 下拉选择患者，关联患者信息，显示患者名称， 产品名称下拉，选择要开单的产品，也可以输入条码，显示产品ID，产品名称，可用量，销售价，销售数量，点击添加购物车，添加销售商品到，单据清单。页面同时显示发票号，总合计，支付金额，支付方式，应找零，点击结算，打印发票。点击新增加添加新销售单据，勾选打印预览，结算或者保存时预览发票。选择发票清单按钮，选择需要关联的 发票信息，搜索发票时，可以指定开始时间，和结束时间检索发票。
    - 前置：
      - 已存在患者信息和产品信息。
    - 输入：
      - 患者编号或选择患者。
      - 选择产品（可通过下拉选择或输入条码）。
      - 销售数量、支付方式等。
    - 输出：
      - 销售单页面，显示患者信息、产品信息、销售数量、价格、支付方式等。
      - 可打印发票。
3.5 报表
    1. 选择患者关联患者编号和患者信息，下拉选择需要查询的产品信息，制定开始时间，和结束时间，从以下几个过滤器进行筛选查询并预览打印报表：
      1. 按日期筛选
      2. 按日期和患者ID筛选
      3. 按日期和客户名称筛选
      4. 按日期和已付账单筛选
      5. 按日期到期付款筛选
      6. 按日期和项目名称筛选
    - 前置：
      - 已存在销售记录。
    - 输入：
      - 筛选条件（如日期、患者ID、产品信息、已付账单等）。
    - 输出：
      - 报表页面，显示符合条件的销售记录。
      - 可预览并打印报表。
## 4. 治疗诊断
4.1 牙周筛查
    1. 新建牙科筛查病例信息，生成接诊ID，就诊日期，就诊时间，预约挂号号码。
    2. 选择患者信息，关联患者名称，年龄，性别
    3. 恒牙检查，标定32颗恒牙，选需要标定的任意牙齿，计数显示牙齿牙位序号和名称，可用重置。
    4. 乳牙标定，选择10颗乳牙，标定的任意牙齿，计数显示牙齿牙位序号和名称，可用重置。
    5. 牙齿网格标定，可用手动输入牙位，选择恒牙或者主要进行
    6. 当选择牙齿后，可以进行牙齿症状评估，手术，和详细诊断医嘱
      1. 症状
        1. 常规性症状，医生描述当前牙齿的基本情况
        2. 既往牙科病史，医生记录是否有牙齿既往病症
        3. 既往病史，医生描述患者是否患有其它疾病
        4. 评估（进行单选项目评估）
          1. 牙齿需要保健
          2. 牙龈有疾病
          3. 需要患者注意牙齿坏习惯
          4. 牙结石，牙菌斑和蛀牙
          5. 例行常规检查
          6. 需要口腔癌症检测
          7. 需要整畸美学
          8. 偶发
          9. 新开发
      2. 手术
        1. 选择手术按钮，选择或者创建手术计划
        2. 选择手术项目，添加到执行计划程序，关联需要执行要处理的牙齿，并备注
      3. 诊断
        1. 选择疾病，创建疾病病因详细描述，或者选择原来病因。
        2. 医生填写诊断结果，可以多项目添加疾病的终端结果
4.2 保存更改
      1. 进行上述检查时，可以随时保存更改信息
4.3 计费
      1. 点击计费，创建账单信息显示牙科筛查需要的费用，打印账单预览个保存
        1. 小计
        2. 折扣
        3. 税费
        4. 总计
        5. 支付金额
        6. 应支付
4.4 患者形态
      1. 显示患者本次病例检查的详细报表
4.5 病假证明
      1. 点击开具病假证明，进行证明开具，然后进行保存并打印
        1. 就诊号
        2. 应用编号
        3. 申请日期
        4. 患者姓名
        5. 年龄
        6. 日期从开始，到结束
        7. 描述患者的具体病因
4.6 预约
      1. 标记患者下一次复查的预约时间
    
    - 前置：
      - 已存在患者信息和医生信息。
    - 输入：
      - 患者信息（通过选择患者获取）。
      - 恒牙和乳牙标定情况。
      - 牙齿症状评估、手术和诊断医嘱。
    - 输出：
      - 牙周筛查结果页面，显示患者信息、筛查结果、评估、手术和诊断医嘱等。
      - 可生成并打印账单，显示费用明细。
      - 可开具病假证明，并保存打印。
      - 可标记患者下一次复查的预约时间。
4.7 病假列表
    1. 指定参数和时间段选择筛选患者的检查报表，可编辑和删除
    - 前置：
      - 已存在病假记录。
    - 输入：
      - 筛选条件（如时间段、患者姓名等）。
    - 输出：
      - 病假列表页面，显示符合条件的病假记录。
      - 可编辑和删除病假记录。
4.8 报表
    1. 选择指定时间段导出检查的详细报表
    - 前置：
      - 已存在牙周筛查记录。
    - 输入：
      - 筛选条件（如时间段等）。
    - 输出：
      - 报表页面，显示符合条件的牙周筛查记录。
      - 可预览并打印报表。
  
## 5. 处方管理
5.1 处方信息
    1. 医生选择患者关联患者信息，创建处方清单项目
    2. 选择药品下拉，备注医嘱，用药周期，数量和备注
    3. 保存更改或者打印处方清单
    - 前置：
      - 已存在患者信息和药品信息。
    - 输入：
      - 患者信息（通过选择患者获取）。
      - 药品信息（通过下拉选择）。
      - 医嘱、用药周期、数量和备注。
    - 输出：
      - 处方单页面，显示患者信息、药品信息、医嘱等。
      - 可打印处方清单。
5.2 处方列表
    1. 指定参数和筛选条件，从开始时间至结束时间筛选 已经开具的处方信息
    2. 编辑和删除处方信息
    - 前置：
      - 已存在处方记录。
    - 输入：
      - 筛选条件（如时间段、患者姓名等）。
    - 输出：
      - 处方列表页面，显示符合条件的处方记录。
      - 可编辑和删除处方记录。
  
## 6. 费用管理
6.1 科目管理
    1. 创建收入或者支出的财务科目，并生成记账凭证
    - 前置：
      - 无特定前置条件。
    - 输入：
      - 科目名称、类型（收入/支出）。
    - 输出：
      - 科目成功添加至系统，生成科目列表页面。
6.2 凭证信息
    2. 选择账户，添加创建费用科目，确定是收入还是支出
    3. 输入金额和选择付款方式，付款方式支持现金和挂账
    4. 指定凭证日期，添加备注，保存创建记账凭证
    - 前置：
      - 已存在科目信息。
    - 输入：
      - 账户信息。
      - 费用科目、金额、付款方式（现金/挂账）。
      - 凭证日期和备注。
    - 输出：
      - 记账凭证页面，显示科目、金额、付款方式等。
6.3 凭证列表
    1. 选择指定时间段开始和结束搜索凭证列表，编辑，打印和删除凭证。
    - 前置：
      - 已存在凭证记录。
    - 输入：
      - 筛选条件（如时间段、账户名称等）。
    - 输出：
      - 凭证列表页面，显示符合条件的凭证记录。
      - 可编辑、打印和删除凭证。
6.4 报表
    1. 按记账日期查找
    2. 按日期撤账户名称查找
    3. 选择指定的账户科目名称查找
    4. 指定时间段开始和结束打印预览
    - 前置：
      - 已存在凭证记录。
    - 输入：
      - 筛选条件（如记账日期、账户名称、科目名称等）。
    - 输出：
      - 报表页面，显示符合条件的凭证记录。
      - 可预览并打印报表。

7.1 临床信息

    - 前置：

    - 输入：

    2. 用户角色有，管理员，医生，收银，财务，资产管理

    - 输入：

    - 前置：
      - 已存在用户信息。

    - 输出：
      - 用户权限设置页面，显示已为用户分配的权限。
